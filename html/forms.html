making things interactive
inputs are important, some include:

  • signup and logins for websites
  • personal info
  • filtering content (dropdowns, checkboxes)
  • searching
  • uploading files
to accomodate these needs, html providese interactive form controls:
  • text inputs (one or multiple lines)
  • radio buttons
  • checkboxes
  • dropdowns
  • upload widgets
  • submit buttons
these controls use different tags, but most of them use the <input> tag.
<input> is a self-closing element. the type of input is defined by its type attribute

for example:

Form Element:
block level element that defines an interactive part of a page.
all form controls(like <input>, <textarea>, or <button>) must appear within a <form> element.
Two html attributes are required:
  • action contains an address that defines where the form info will be sent.
  • method can be either GET or POST and defines how the form info will be sent
form info is sent to a server.
think of a form as a collection of input controls that work together to perform a single operation.
if you wrote a login form, you would have 3 controls:
  • an email input (<input type="email">
  • a password input (<input type="password">
  • a submit button (<input type="submit">
These 3 elements would be enclosed within a single form

Text Inputs:
p much all forms require text input from the user in order for them to enter info,
Text form controls come in different variations:
  • text (input type="text") --> allows any type of character
  • email (input type="email") --> might display warning if an invalid email is entered
  • password (input type="password) --> shows dots as characters
  • number (input type=number") up/down keboard keys can be used
  • telephone (input type="tel") can trigger an autofill
  • multiple line text(<textarea> </textare>) can be resized

Placeholderss:
text inputs can display a placeholder text that disappears as soon as text is entered:
<input type="text" placeholder="Enter your name">

Labels:
because form elements aren't descriptive, they are usually preceded by a label:
<label> </label>
While placeholders already provide some hint at what content is expected, labels have the advantage
of remaining visible at all times, and can be used alongside other types of form controls, like checkboxes
or radio buttons.

you could theoretically use <p> to describe form elements, but using <label> is semantically more
valid because they only exist within forms.
you can make a label for an input attribute (for=input id)

Checkboxes:
only 2 states, checked or unchecked.
yes or no stuff
(input type=checkbox)
wrap it in a label tag because its hard to click on the box sometimes
you can make it checked by default by adding the checked attribute

Radio Buttons:
presenting the user with a list of options to choose from
For this form control to work, your code needs to group a list of buttons together
aka use the same value for the name attribute
(input type=radio name=(group name))
radio buttons are said to be mutually exclusive.

The difference between radio buttons and checkboxes:
checkboxese exist on their own, radio buttons can only appear as a list (@ least 2 options)
also, clicking a checkbox is optional. This is why it's impossible to uncheck a radio button unless choosing a sibling option.
choosing a radio button is mandatory

Dropdown Menus:
If the number of options to choose from takes up too much space on the page, use the <select> tag for dropdown menus.
<option> should be the child tag of <select>
Multiple choice dropdowns:
add the multiple attribute to <select> (hold down ctrl and click for multiple choices)
good alternative for multi checkboxes
